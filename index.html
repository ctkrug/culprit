Analyze a HAR file online, without uploading it

Culprit is a website waterfall analyzer that reads a .har network export and tells you which requests are slowing the page down. Most tools stop at a waterfall chart and leave the reading to you. Culprit does the reading: it ranks every image, script, font, and third-party tracker by how much load time it costs, and writes a one-line fix for the worst offenders. Everything runs in your browser, so the file never leaves your machine.

Frequently asked questions

How do I analyze a HAR file online?
Export a .har file from your browser's network panel, then drop it onto the page above. Culprit parses it locally and returns a ranked punch list in about a second. There is no sign-up and no upload step.
Is Culprit a HAR file viewer?
It is a HAR file viewer with an opinion. A plain viewer replays the waterfall you already saw in DevTools. Culprit reads that same data and points at the specific requests worth fixing first, with the byte size and time cost for each.
What is a website waterfall analyzer?
A waterfall analyzer looks at the timeline of every request a page makes and works out where the load time actually goes. Culprit adds the interpretation step: it splits first-party from third-party bytes, names the single largest host, and flags render-blocking scripts and oversized images by name.
Does my HAR file get uploaded anywhere?
No. Parsing happens entirely in your browser with client-side JavaScript. HAR files often contain internal URLs and header data, so keeping the file on your machine is the only safe default. You can confirm this by reading the source on GitHub.
How do I capture a HAR file to analyze?
In Chrome or Edge, open DevTools, go to the Network tab, reload the page, then right-click any request and choose "Save all as HAR." Firefox and Safari have the same export in their network panels. Culprit reads exports from all three.
